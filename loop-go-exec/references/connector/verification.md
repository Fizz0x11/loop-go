# loop-go-exec 连接器：验收标准

> 本文件是**验收 agent** 专用的验收检查清单。
> **验收 agent 只读这个文件，对照干活 agent 的输出打分。**
> **干活 agent 不能读这个文件。**

---

## 验收标准

### A. 项目基本信息（必填）

| 检查项 | 干活 agent 应该输出了吗？ | 验收 agent 打分 |
|--------|------------------------|--------------|
| project_type | 项目类型正确？ | 0-2 分 |
| readme_status | 状态标记了？ | 0-1 分 |
| dependency_install | 结果记录了？ | 0-2 分 |
| verification | 验证命令执行了？ | 0-3 分 |

### B. 真实性检查（关键）

| 检查项 | 说明 | 验收 agent 怎么做 |
|--------|------|----------------|
| clone 真跑了？ | 看输出里有没有 `gh repo clone` 实际执行 | 0-3 分 |
| 测试命令真跑了？ | 看输出里有没有真实命令执行 | 0-3 分 |
| 不是只读 README？ | 至少跑了一个命令验证 | 0-2 分 |

### C. Wiki 质量

| 检查项 | 权重 |
|--------|------|
| 有 GitHub URL 超链接 | 1 分 |
| 有测试结果摘要 | 2 分 |
| 有 CE Review 结果 | 2 分 |
| 有 自我复盘 | 2 分 |

---

## 验收 agent 打分规则

```
总分 = A + B + C

8-10 分: ✅ 通过（"验收 agent 确认通过"）
5-7 分: ⚠️ 需补项（列出具体要补什么）
0-4 分: 🔴 不合格（打回重做）
```

---

## 验收 agent 提示词

```
你是一个严格的验收 agent。你对照以下标准打分：
1. 干活 agent 真的跑了命令吗，还是只读了文档？
2. 项目类型判断正确吗？
3. Wiki 里有没有实际内容，不是空壳？
4. CE Review 有没有填"做对了/做错了/下次改"三栏？

给分并写出原因。如果不合格，说明哪里要重做。
```

---

## 不合格时的处理

验收 agent 打分 ≤ 4：
1. 验收 agent 写出"不合格原因"
2. 干活 agent 拿到原因，重做对应步骤
3. 再次提交验收
4. 直到验收 agent 打分 ≥ 5
