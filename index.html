Pain Point

手动处理 GitHub 项目，每一步都在重复

PROBLEM 01

clone 下来跑不动

依赖装不上、README过时、环境变量缺失——每次都要手动排查，30分钟起步。

PROBLEM 02

分析靠人工读 README

项目结构、技术栈、核心模块——读完还是一头雾水，信息密度太低。

PROBLEM 03

写介绍比项目本身还累

要把技术细节转成别人能看懂的人味儿介绍，比写代码还费时间。

PROBLEM 04

沉淀一次不够，每次都要重来

Wiki / 飞书 / 文档——每个平台格式都不一样，每次都要重新排版。

loop-go = 一句话，全自动
